# 75HER Challenge: Decision Log
**Project Name:** Planet Hop | **Team Name:** Jess & Steph

## 🛠 Decision Log

| Category | Decision → Why | Tradeoff |
|----------|----------------|----------|
| **Tech Stack** | Three.js → Chosen for 3D rendering capabilities and extensive VR support | Larger bundle size (~600KB) but enables cross-platform VR experience |
| **Tech Stack** | Vite → Selected as build tool for fast development and modern ES modules | Requires Node.js 18+ but provides superior dev server performance |
| **Tech Stack** | Howler.js → Used for audio management with Web Audio API support | Additional dependency but handles iOS Safari audio restrictions |
| **Architecture** | Static Web Application → No backend server, runs entirely in browser | Limited data persistence but enables instant deployment and zero server costs |
| **Architecture** | Modular Component Design → Separated scene setup, planet factory, UI panels, and story data | More files to manage but improves maintainability and team collaboration |
| **Feature Scope** | WebXR Support → Implemented VR functionality for immersive experience | Increased complexity but provides unique educational value |
| **Feature Scope** | Bilingual Support → Added English/Spanish language detection and switching | Double content creation but expands accessibility to wider audience |
